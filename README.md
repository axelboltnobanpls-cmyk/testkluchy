# 🤖 Бот для бесплатных ключей — CopperShop (это тест, для заказчика)

Полностью рабочий Telegram-бот, который выдаёт бесплатные ключи за подписку на канал.  
Акция одноразовая — один ключ на человека.

---

## 📁 Структура проекта

```
CopperS_bot/
├── .python-version   — версия пайтона
├── bot.py            — основной файл бота
├── keys.json         — база ключей (загружаете сюда свои)
├── users.db          — база данных пользователей (создаётся автоматически)
├── config.json       — конфигурация (сохраняет канал)
├── requirements.txt  — зависимости Python
└── README.md         — эта инструкция
```

## 🧰 Шаг 1: Подготовка

### 1.1 Установите Python
Скачайте и установите Python 3.10+ с официального сайта:  
👉 https://www.python.org/downloads/  
При установке **обязательно** поставьте галочку **«Add Python to PATH»**.

### 1.2 Создайте бота у @BotFather
1. Откройте Telegram и найдите **@BotFather**
2. Отправьте `/newbot`
3. Задайте имя бота (например: `Copper Keys Bot`)
4. Задайте юзернейм (например: `coppers_keys_bot`)
5. Скопируйте полученный **токен** — он выглядит примерно так:  
   `7123456789:AAF123AbCdEfGhIjKlMnOpQrStUvWxYz`

### 1.3 Сделайте бота администратором канала
1. Откройте канал **@coppers_shop** (или создайте новый)
2. Зайдите в **Настройки канала → Администраторы**
3. Добавьте вашего бота и дайте ему права **администратора**  
   (минимум: права на чтение сообщений и список участников)  
   ⚠️ **Без прав администратора проверка подписки НЕ будет работать!**

## 🛠️ Шаг 2: Настройка кода

### 2.1 Откройте файл `bot.py` в любом текстовом редакторе
Рекомендуется: **VS Code** (бесплатный) или **Notepad++**

### 2.2 Измените настройки (в начале файла)

```python
BOT_TOKEN = "ВСТАВЬТЕ_ВАШ_ТОКЕН_СЮДА"      # Токен от BotFather
CHANNEL_USERNAME = "@coppers_shop"            # Юзернейм вашего канала
ADMIN_IDS = [123456789]                       # Ваш Telegram ID (число)
```

**Как узнать свой Telegram ID?**  
Напишите боту **@userinfobot** в Telegram — он покажет ваш ID.

## 📦 Шаг 4: Загрузка ключей

### 4.1 Через файл `keys.json`
Откройте файл `keys.json` и запишите свои ключи, каждый в кавычках, через запятую:

```json
{
  "keys": [
    "ABCD-1234-EFGH-5678",
    "IJKL-9012-MNOP-3456",
    "QRST-7890-UVWX-1234"
  ]
}
```

### 4.2 Через Telegram (команда админа)
Отправьте боту в личку:
```
/addkey
XXXXX-XXXXX-XXXXX
YYYYY-YYYYY-YYYYY
```
Бот добавит ключи в базу и покажет сколько теперь доступно.

### Как работает система ключей:
- Ключи хранятся в файле `keys.json` (порядок — очередь FIFO)
- При выдаче ключ **удаляется** из файла → никто больше не получит его
- Все выданные ключи записываются в `users.db` с привязкой к пользователю
- Даже при перезапуске бота ключи не дублируются — используется SQLite

## 👤 Как это работает для пользователя

### Сценарий 1: Пользователь НЕ подписан
1. Пользователь нажимает `/start`
2. Бот пишет: «Для получения ключа подпишитесь на канал» + показывает кнопку
3. Пользователь нажимает кнопку → переходит в канал → подписывается
4. Возвращается в бота → нажимает «✅ Проверить подписку»
5. Бот проверяет подписку → если всё ОК → выдаёт ключ

### Сценарий 2: Пользователь уже подписан
1. Пользователь нажимает `/start`
2. Бот мгновенно проверяет подписку → видит что подписан → **сразу выдаёт ключ**
3. Пользователь получает ключ + инструкцию

### Сценарий 3: Ключи закончились
1. Пользователь нажимает `/start`
2. Бот пишет: «Ключи закончились, следите за обновлениями»

## 🔧 Админские команды

Все команды отправляются боту **в личные сообщения** (не в канал!):

| Команда | Описание | Пример |
|---------|----------|--------|
| `/addkey` | Добавить ключи (каждый на новой строке после команды) | `/addkey\nABC-123\nDEF-456` |
| `/delkey` | Удалить конкретный ключ из базы | `/delkey ABC-123` |
| `/listkeys` | Показать все доступные ключи | `/listkeys` |
| `/stats` | Статистика: сколько выдано ключей, сколько осталось | `/stats` |
| `/setchannel` | Сменить канал для проверки подписки | `/setchannel @new_channel` |
| `/start` | Старт (для пользователей) | `/start` |

## 🖥️ Шаг 5: Бесплатный хостинг (чтобы бот работал 24/7)

### Вариант 1: **Railway.app** ⭐ (Рекомендуется)
- 🔗 https://railway.app
- Бесплатный тариф: 500 часов/месяц (хватит для 1 бота)
- Поддерживает Python, подключается к GitHub
- Бот работает 24/7

**Как развернуть:**
1. Создайте аккаунт на railway.app (можно через GitHub)
2. Создайте новый проект → «Deploy from GitHub Repo»
3. Загрузите этот проект на GitHub (или создайте пустой репо и загрузите файлы)
4. В настройках проекта добавьте переменную окружения:  
   `BOT_TOKEN=ваш_токен`
5. В файле `bot.py` замените строку `BOT_TOKEN = "ВАШ_ТОКЕН"` на:  
   ```python
   BOT_TOKEN = os.environ.get("BOT_TOKEN", "ВАШ_ТОКЕН")
   ```
6. Добавьте в начало файла: `import os`
7. Нажмите Deploy — бот запустится!

## ⚡ Дополнительно

### Как добавить фото к инструкции
Чтобы отправить фото с текстом, замените блок с ключом на:

```python
await bot.send_photo(
    chat_id=callback.from_user.id,
    photo=open("screenshot.png", "rb"),
    caption=f"🎉 Вот ваш ключ:\n<code>{key}</code>\n\n📖 Инструкция:\n..."
)
```

Поместите файл `screenshot.png` рядом с `bot.py`.

### Файл `users.db`
Это SQLite база данных. Все выданные ключи записаны там.  
Чтобы посмотреть — откройте файл через любой SQLite-браузер или DB Browser for SQLite.

## ❓ Частые вопросы

**В:** Бот не проверяет подписку — что делать?  
**О:** Убедитесь что бот добавлен как **администратор** в канал, а не просто участник.

**В:** Ключи заканчиваются, а пользователей много?  
**О:** Загружайте ключи через `/addkey` в личку бота.

**В:** Можно ли несколько каналов?  
**О:** Да! Используйте команду `/setchannel @channel1 @channel2` (нужно доработать код).
